---
title: Quick start
description: Create your first Flock workspace and run agents in persistent terminal panes.
---

Start Flock from any project directory:

```bash
flock
```

Flock launches or attaches to your default background session. You do not manage sockets. If you detach, agents keep running.

For the local, SSH, and `flock --remote` workflows, see [How to work with Flock](/docs/how-to-work/).

## Create a workspace

Press `prefix+shift+n` to create a workspace. A workspace is a project-level container for tabs, panes, and agents.

Give each active project its own workspace. This keeps agent state readable in the sidebar.

## Run an agent

Start your agent in the root pane.

```bash
pi
```

Flock detects supported agents automatically. The sidebar shows whether each agent is `working`, `blocked`, `done`, or `idle`.

## Keyboard control

Press `ctrl+b` to enter prefix mode, then press an action key.

Common actions:

| Action | Key |
| --- | --- |
| Split right | `prefix+v` |
| Split down | `prefix+minus` |
| New tab | `prefix+c` |
| Next / previous tab | `prefix+n` / `prefix+p` |
| Workspace navigation | `prefix+w` |
| New workspace | `prefix+shift+n` |
| Detach client | `prefix+q` |

After detaching, run `flock` again to reattach to the same session.

## Use the mouse

Flock is mouse-native. You can click panes, tabs, workspaces, and agents; drag borders; drag-select text to copy it to your clipboard; double-click a token to copy it directly; and use right-click menus. Copying does not require Ctrl+C.

If you configure `ui.right_click_passthrough_modifier`, that modifier plus right-click sends right-click, hold, and drag gestures to mouse-reporting pane apps.

Ctrl-click opens pane links when your terminal sends the modified click to Flock. This works for OSC 8 hyperlinks and visible `http://` or `https://` URLs. The portable terminal-native fallback is Shift-Ctrl-click on Linux or Shift-Cmd-click on macOS.

## Copy from the keyboard

Press `prefix+[` to enter copy mode for the focused pane. Use `h/j/k/l`, `w/b/e`, and `{`/`}` to move, `v` or Space to start a selection, `y` or Enter to copy it, and `q` or Esc to leave without copying.

## Manage named sessions

Named sessions are separate Flock server namespaces. Use them when you want fully separate runtime state.

```bash
flock session list
flock session attach work
flock session attach side-project
flock session stop work
flock session delete side-project
```

Workspaces are usually enough. Use named sessions when you need isolation between sets of panes, sockets, and persistent state.

## Attach from another machine

Run Flock where the work lives. If the code and credentials are on a server, either SSH there and run Flock like a terminal multiplexer:

```bash
ssh you@server
flock
```

Or attach from your local terminal through SSH:

```bash
flock --remote workbox
flock --remote ssh://you@server:2222
```

For repeat targets, put the host in your SSH config. See [How to work with Flock](/docs/how-to-work/) for the full model.
