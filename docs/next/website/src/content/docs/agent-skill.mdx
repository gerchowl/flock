Flock ships a reusable agent skill file at [`SKILL.md`](https://github.com/gerchowl/flock/blob/master/SKILL.md).

Install that file into any coding agent that supports reusable skills or custom instructions. The skill teaches the agent how to control Flock from inside a Flock pane.

## What the skill does

The skill tells an agent to use the `flock` CLI when `FLOCK_ENV=1` is set. That means the agent is running inside a Flock-managed pane and can safely talk to the local Flock socket.

With the skill installed, an agent can:

- inspect workspaces, tabs, panes, and neighboring agents
- split panes and run commands without stealing focus
- read pane output and recent logs
- wait for servers, tests, or another agent to finish
- start helper agents in sibling panes

The skill is not a separate app or service. It is a markdown instruction file for agents.

## Install it

Use the repository copy as the source of truth:

```text
https://github.com/gerchowl/flock/blob/master/SKILL.md
```

For agents with a skill system, install that file as a skill named `flock`. For agents without a skill system, paste the file into the agent's project or user instructions.

After installation, start the agent inside Flock:

```bash
flock
claude
```

Or use any other coding agent in a Flock pane. The important part is that the agent process runs inside Flock, so `FLOCK_ENV=1` is available.

## Safety rule

The skill starts with one guardrail: if `FLOCK_ENV=1` is not set, the agent should stop and say it is not running inside a Flock-managed pane.

This prevents an agent outside Flock from trying to control a session it does not own.
