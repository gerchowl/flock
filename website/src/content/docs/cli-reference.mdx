Flock’s CLI talks to the running server over the same local socket API used by integrations and agents.

Most commands print JSON responses. Use them from scripts when you want deterministic automation.

## Launch and status

```bash
flock                         # launch or attach to the default session
flock --session work          # launch or attach to a named session
flock --remote workbox        # attach through SSH, using local keybindings
flock --remote workbox --remote-keybindings server
flock --remote workbox --handoff
flock --no-session            # single-process escape hatch
flock --default-config        # print default config
flock update                  # download and install from the configured channel
flock update --handoff        # opt into live handoff for supported running servers
flock channel show            # print stable or preview
flock channel set preview     # opt into preview builds
flock channel set stable      # return to stable releases
flock --version               # print version
```

Status commands:

```bash
flock status
flock status server
flock status client
```

## Server

```bash
flock server
flock server stop
flock server reload-config
```

`flock server` runs the headless server explicitly. Use it for supervised or service-style setups. `reload-config` applies reloadable settings without restarting panes.

## Sessions

```bash
flock session list [--json]
flock session attach <name>
flock session stop <name> [--json]
flock session delete <name> [--json]
```

Use `default` as the session name when you need to stop the default session explicitly.

## Workspaces

```bash
flock workspace list
flock workspace create [--cwd PATH] [--label TEXT] [--focus] [--no-focus]
flock workspace get <workspace_id>
flock workspace focus <workspace_id>
flock workspace rename <workspace_id> <label>
flock workspace close <workspace_id>
```

Create a workspace without stealing focus:

```bash
flock workspace create --cwd ~/project --label api --no-focus
```

## Worktrees

```bash
flock worktree list [--workspace ID | --cwd PATH] [--json]
flock worktree create [--workspace ID | --cwd PATH] [--branch NAME] [--base REF] [--path PATH] [--label TEXT] [--focus] [--no-focus] [--json]
flock worktree open [--workspace ID | --cwd PATH] (--path PATH | --branch NAME) [--label TEXT] [--focus] [--no-focus] [--json]
flock worktree remove --workspace ID [--force] [--json]
flock worktree kill --workspace ID [--dry-run] [--force] [--keep-branch] [--json]
```

Worktrees are normal Flock workspaces with Git checkout provenance. `worktree create` creates a Git worktree checkout, opens it as a workspace, and groups it with the parent repo workspace. Without `--path`, Flock creates the checkout under `<worktrees.directory>/<repo>/<branch-slug>`.

`workspace close` closes Flock state only. `worktree remove` is the explicit checkout deletion path; it runs `git worktree remove`, never deletes the branch, and requires `--force` when Git refuses a dirty checkout.

`worktree kill` is the scriptable twin of the TUI's "Kill worktree & branch": it runs the same merge gate (merged PR via `gh`, merged into the default branch, or remote containment) and deletes the local branch only with positive evidence. `--dry-run` prints the gate verdict as JSON (exit 0 evidence found, 3 none); `--keep-branch` always preserves the branch; dirty checkouts exit 4 until `--force`. Agents can use this as the single entry point for post-merge cleanup.

## Panes

```bash
flock pane list [--workspace <workspace_id>]
flock pane get <pane_id>
flock pane rename <pane_id> <label>|--clear
flock pane split <pane_id> --direction right|down [--cwd PATH] [--focus] [--no-focus]
flock pane close <pane_id>
```

Read output:

```bash
flock pane read <pane_id> [--source visible|recent|recent-unwrapped] [--lines N]
flock pane read <pane_id> --source visible --ansi
flock pane read <pane_id> --source recent-unwrapped --lines 120
```

Send input:

```bash
flock pane send-text <pane_id> <text>
flock pane send-keys <pane_id> <key> [key ...]
flock pane run <pane_id> <command>
```

`pane run` submits text plus Enter atomically. Prefer it over `send-text` plus `send-keys Enter` for commands.

Report agent state from custom hooks:

```bash
flock pane report-agent <pane_id> \
  --source ID \
  --agent LABEL \
  --state idle|working|blocked|unknown \
  [--message TEXT] \
  [--custom-status TEXT] \
  [--seq N] \
  [--agent-session-id ID] \
  [--agent-session-path PATH]
```

`pane get`, `pane list`, `agent get`, and `agent list` include a read-only `agent_session` object when an official integration has reported a native session reference. If no native session reference is stored, the field is omitted.

Those commands include `foreground_cwd` when Flock can resolve the cwd of the foreground process controlling the pane. The existing `cwd` field remains the pane/workspace cwd used for labels and follow-cwd behavior.

Report display-only pane metadata without taking over semantic state:

```bash
flock pane report-metadata <pane_id> \
  --source ID \
  [--agent LABEL] \
  [--applies-to-source ID] \
  [--title TEXT|--clear-title] \
  [--display-agent TEXT|--clear-display-agent] \
  [--custom-status TEXT|--clear-custom-status] \
  [--state-label STATUS=TEXT] \
  [--clear-state-labels] \
  [--seq N] \
  [--ttl-ms N]
```

`STATUS` is one of `idle`, `working`, `blocked`, `done`, or `unknown`. `--agent` is a guard for the authoritative agent label. `--applies-to-source` is a guard for the active lifecycle authority source. Use `--display-agent` to change the visible name.

Promote session-specific fields (containers, progress, custom KV) into the calling pane's header:

```bash
flock pane set-field <key> <value> [--ttl <secs>] [--pane <pane_id>]
flock pane clear-field <key> [--pane <pane_id>]
```

Fields render as compact `key value` chips on the pane header context line (after the branch/PR segments) and on the navigation surfaces — sidebar agent panel rows, navigator pane lists, mobile pane lists. The pane defaults to the calling pane (`$FLOCK_PANE_ID`, healed by process ancestry), so hooks and wrapper scripts inside a pane need no bookkeeping:

```bash
# a podman wrapper registering a dev container
flock pane set-field pg "● up"
# a build script publishing progress that auto-expires if it stalls
flock pane set-field build 73% --ttl 120
flock pane clear-field build
```

Fields are display-only and ephemeral — never persisted into session snapshots. Caps: 6 fields per pane, key up to 16 characters, value up to 48 characters; over-cap requests fail. Setting an existing key updates it in place.

## Agents

```bash
flock agent list
flock agent get <target>
flock agent read <target> [--source visible|recent|recent-unwrapped] [--lines N] [--format text|ansi] [--ansi]
flock agent send <target> <text>
flock agent rename <target> <name>|--clear
flock agent focus <target>
flock agent wait <target> --status <idle|working|blocked|unknown> [--timeout MS]
flock agent attach <target> [--takeover]
flock agent start <name> [--cwd PATH] [--workspace ID] [--tab ID] [--split right|down] [--focus|--no-focus] -- <argv...>
```

Agent targets can be terminal IDs, unique agent names, detected or reported agent labels, or legacy pane IDs. Names and labels are agent identities. Terminal IDs and legacy pane IDs are low-level escape hatches.

`agent read` reads the resolved terminal stream. `agent send` writes literal text to that stream. `agent get`, `agent focus`, `agent wait`, and `agent attach` require the resolved terminal to have agent identity. `agent rename` can assign that identity.

Use `pane send-text`, `pane send-keys`, `pane run`, and `terminal attach` for ordinary terminals, servers, tests, shells, or low-level terminal control. Use `pane run` when you want to submit a command with Enter.

## Direct terminal attach

```bash
flock terminal attach <terminal_id> [--takeover]
```

Detach from direct attach with `ctrl+b q`. Send literal `ctrl+b` with `ctrl+b ctrl+b`.

## Waits

Wait for output in a pane:

```bash
flock wait output <pane_id> --match <text> [--source visible|recent|recent-unwrapped] [--lines N] [--timeout MS] [--regex] [--raw]
```

Wait for pane agent state:

```bash
flock wait agent-status <pane_id> --status <idle|working|blocked|done|unknown> [--timeout MS]
```

Use `wait output` for normal commands and servers. Use `wait agent-status` for coding agents.

## Read sources

| Source | Meaning |
| --- | --- |
| `visible` | Current rendered screen. Best for UI feedback loops. |
| `recent` | Recent scrollback with terminal wrapping. |
| `recent-unwrapped` | Recent scrollback without soft wrapping. Best for logs. |

## Environment variables

| Variable | Purpose |
| --- | --- |
| `FLOCK_CONFIG_PATH` | Override the config file path. |
| `FLOCK_SESSION` | Select a named session for CLI commands. |
| `FLOCK_SOCKET_PATH` | Low-level socket path override. |
| `FLOCK_LOG` | Set log filter, for example `FLOCK_LOG=flock=debug`. |
| `FLOCK_PANE_ID` | Baked into every pane's environment; the default pane for `pane set-field`/`clear-field`. Stale values are healed by process ancestry. |
| `FLOCK_<UPPER_SNAKE_PATH>` | Generic override for any scalar config field (ADR-0002); sits BELOW the config file. |
| `FLOCK_DISABLE_SOUND` | Deprecated alias for `FLOCK_UI_SOUND_ENABLED=false`. Sits below the file (removal after one release). |
