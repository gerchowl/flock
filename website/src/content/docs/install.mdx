---
title: Install Flock
description: Install, update, and verify Flock on Linux and macOS.
---

Flock ships as a single binary for Linux and macOS.

## Install

Run the installer:

```bash
curl -fsSL https://flock.dev/install.sh | sh
```

The installer downloads the right release binary for your platform and places it on your PATH.

## Install with Homebrew

If you already use Homebrew:

```bash
brew install flock
```

## Install with mise

If you already use mise:

```bash
mise use -g flock
```

## Verify

Start Flock:

```bash
flock
```

If your shell cannot find `flock`, restart the terminal or check that the install directory is on your PATH.

## Update

Flock checks for new releases and notifies you in the app. You can update manually:

```bash
flock update
```

`flock update` is for installs managed by Flock's own installer. Homebrew, mise, and Nix installs are updated through those package managers instead.

Flock uses the stable update channel by default. To opt into preview builds from `master`, set the channel:

```bash
flock channel set preview
```

Switch back to stable the same way:

```bash
flock channel set stable
```

For direct installs, changing channels also checks that channel and installs its latest binary. If that update fails, run `flock update` to retry from the configured channel.

Preview builds are GitHub prereleases published from the current development branch. They are useful when you want fixes before the next stable release, but they can regress. Homebrew, mise, and Nix installs do not use the preview channel.

By default, `flock update` installs the new binary and leaves compatible running sessions alone. If an update changes Flock's client/server protocol, Flock asks whether to stop the old server after installing. Stop the old server to use the new version. Stopping exits pane processes. For the default session, run `flock server stop`, then run `flock` again. For a named session, run `flock session stop <name>`, then run `flock session attach <name>` again.

To opt into experimental live server handoff for supported running sessions, run:

```bash
flock update --handoff
```

Live handoff does not apply to Homebrew, mise, or Nix package-manager updates. For those installs, update with the package manager, then restart that Flock session when you are ready to use the new server. If a running session still uses the old server, stop it with `flock server stop` or `flock session stop <name>`, then run Flock again.

## Install with Nix

If you already use Nix, Flock provides a flake that builds Flock from source:

```bash
nix run github:gerchowl/flock/v0.x.y
nix build github:gerchowl/flock/v0.x.y
nix profile install github:gerchowl/flock/v0.x.y
```

Replace `v0.x.y` with the latest release tag. You can omit the tag to track `master`, but release tags are recommended for normal installs.

The flake also exposes a development shell:

```bash
nix develop github:gerchowl/flock
```

Update through the same Nix workflow you used to install Flock. For a profile install, list your profile entries and upgrade the Flock entry:

```bash
nix profile list
nix profile upgrade <index-or-name>
```

If Flock is an input in your own flake, update that input and rebuild your system, Home Manager, or development environment:

```bash
nix flake update flock
```

## Download manually

You can also download a binary from [GitHub releases](https://github.com/gerchowl/flock/releases).

Choose the asset that matches your system:

| System | Asset |
| --- | --- |
| Linux x86_64 | `flock-linux-x86_64` |
| Linux aarch64 | `flock-linux-aarch64` |
| macOS Intel | `flock-macos-x86_64` |
| macOS Apple silicon | `flock-macos-aarch64` |

Make it executable and move it somewhere on your PATH.

```bash
chmod +x flock-linux-x86_64
mv flock-linux-x86_64 ~/.local/bin/flock
```

## Requirements

Flock supports Linux and macOS. Native Windows support is not available yet; use Flock inside WSL for now.
